# Michael Stonebraker: Timeline

## Early Life and Education (1943-1971)
| Year | Event | Significance |
|------|-------|--------------|
| 1943 | Born October 11, New Milton, Hampshire, England | Birth in post-war England |
| 1940s | Family moves to United States | Raised in American technical culture |
| 1965 | B.S. Electrical Engineering, Princeton University | Engineering foundation |
| 1966 | M.S. Electrical Engineering, University of Michigan | Graduate studies begin |
| 1971 | Ph.D. Computer Information and Control Engineering, Michigan | Doctoral work completed; joins UC Berkeley |

## The Ingres Era (1971-1985)
| Year | Event | Significance |
|------|-------|--------------|
| 1971 | Joins UC Berkeley as Assistant Professor | Academic career begins |
| 1973 | Attends Codd lecture on relational model | Convinced of relational database viability |
| 1974 | Begins Ingres project with Eugene Wong | First major database system |
| 1976 | "Design and Implementation of INGRES" published | Foundational paper establishes relational practicality |
| 1980 | Founds Relational Technology, Inc. (Ingres Corporation) | First academic database commercialization |
| 1984 | Ingres Corporation achieves commercial success | Validation of academic-to-commercial model |
| 1985 | Ingres acquired by ASK Computer Systems | First exit; begins planning next system |

## The Postgres Era (1985-2000)
| Year | Event | Significance |
|------|-------|--------------|
| 1986 | Begins Postgres project | Object-relational database development |
| 1986 | "The Design of Postgres" published | Architectural vision articulated |
| 1988 | First Postgres prototype operational | Working system demonstrates concepts |
| 1992 | Founds Illustra (Postgres commercialization) | Second startup for object-relational systems |
| 1994 | Postgres development ends at Berkeley | Academic phase concludes |
| 1995 | Illustra acquired by Informix | Second successful exit |
| 1995 | Postgres released as open source | Becomes PostgreSQL project |
| 1997 | Founds Cohera | Third startup (commercial PostgreSQL) |
| 1999 | Cohera acquired by PeopleSoft | Third exit |

## MIT and Specialized Systems (2001-2010)
| Year | Event | Significance |
|------|-------|--------------|
| 2001 | Joins MIT as Adjunct Professor | New institutional home |
| 2003 | Founds StreamBase Systems | Stream processing startup |
| 2005 | Founds Vertica Systems | Column-store startup |
| 2005 | "C-Store" paper published | Column-store architecture established |
| 2007 | "One Size Fits All" paper published | Architectural manifesto |
| 2008 | Publishes MapReduce critique | Controversial but influential position |
| 2009 | Founds VoltDB | In-memory transaction processing |
| 2009 | Founds SciDB | Scientific database array system |

## Recognition and Continued Innovation (2011-Present)
| Year | Event | Significance |
|------|-------|--------------|
| 2011 | Vertica acquired by HP (~$350M) | Largest exit; validates column-stores |
| 2013 | Founds Tamr | Machine learning for data unification |
| 2014 | Receives ACM Turing Award | Highest honor in computing |
| 2014 | "The Seeds of Stonebraker's Turing Award" published | Retrospective analysis of contributions |
| 2015 | Turing Award Lecture delivered | "The Land Sharks of the Database World" |
| 2015 | Receives IEEE John von Neumann Medal | Additional major recognition |
| 2017 | StreamBase acquired by TIBCO | Continued startup success |
| 2020s | Continued MIT research and advisory roles | Ongoing influence on field |

## Era Summary
| Period | Focus | Key Output |
|--------|-------|------------|
| 1971-1985 | Relational databases | Ingres, QUEL, commercial RDBMS foundation |
| 1985-2000 | Object-relational systems | Postgres, PostgreSQL, Illustra |
| 2001-2014 | Specialized databases | StreamBase, Vertica, VoltDB, SciDB |
| 2014-present | Recognition and continued work | Turing Award, ongoing research |
