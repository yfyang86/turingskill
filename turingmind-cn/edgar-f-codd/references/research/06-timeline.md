# Edgar F. Codd - Timeline

## Early Life and Education (1923-1948)

| Year | Age | Event |
|------|-----|-------|
| 1923 | 0 | Born August 19, Fortuneswell, Isle of Portland, England |
| 1930s | 7-17 | Attends Poole Grammar School, excels in mathematics |
| 1942 | 19 | Joins Royal Air Force as pilot |
| 1945 | 22 | WWII service ends, enrolled at Oxford University |
| 1948 | 25 | B.A. Mathematics and Chemistry, Oxford; emigrates to USA |

## IBM Early Career (1948-1965)

| Year | Age | Event |
|------|-----|-------|
| 1948 | 25 | Joins IBM as programming mathematician in New York |
| 1949-1953 | 26-30 | Works on SSEC (Selective Sequence Electronic Calculator) |
| 1957-1960 | 34-37 | Contributes to early FORTRAN compiler development |
| 1961 | 38 | Begins PhD studies at University of Michigan |
| 1965 | 42 | PhD in Computer Science; thesis on cellular automata |

## Relational Model Genesis (1965-1975)

| Year | Age | Event |
|------|-----|-------|
| 1965 | 42 | Returns to IBM, continues database research |
| 1968 | 45 | Moves to IBM San Jose Research Laboratory |
| 1969 | 46 | IBM Research Report RJ599: Early normalization concepts |
| 1970 | 47 | **Seminal CACM paper: "A Relational Model of Data"** |
| 1971 | 48 | Publishes normalization theory (2NF, 3NF) |
| 1972 | 49 | Relational completeness theorem published |
| 1974 | 51 | System R project initiated at IBM San Jose |
| 1974 | 51 | Raymond Boyce dies in hiking accident |

## Recognition and Adoption (1975-1985)

| Year | Age | Event |
|------|-----|-------|
| 1976 | 53 | System R papers published; SQL language emerges |
| 1979 | 56 | Oracle releases first commercial SQL database |
| 1981 | 58 | **Awarded ACM Turing Award** |
| 1982 | 59 | Turing Award lecture published in CACM |
| 1983 | 60 | IBM announces DB2 database product |
| 1984 | 61 | IEEE Centennial Medal awarded |
| 1984 | 61 | **Leaves IBM after 36 years** |
| 1985 | 62 | Publishes "Twelve Rules for Relational Databases" |

## Independent Career (1985-2003)

| Year | Age | Event |
|------|-----|-------|

| 1995 | 72 | Computer History Museum oral history interview |
| 1997 | 74 | Gradual reduction in consulting activities |

## Final Years and Legacy (1998-2003)

| Year | Age | Event |
|------|-----|-------|

## Technological Context Timeline

### Computing Era Context
| Decade | Technology | Database Context |
|--------|-----------|------------------|
| 1950s | Vacuum tubes, magnetic drums | File systems, early databases |
| 1960s | Transistors, disk storage | Hierarchical (IMS), network (CODASYL) |
| 1970s | Integrated circuits, minicomputers | **Relational model proposed (1970)** |
| 1980s | Personal computers, local networks | SQL standardization, PC databases |
| 1990s | Internet, client-server | Web databases, data warehousing |
| 2000s | Cloud computing, big data | NoSQL movement, then NewSQL return |
| 2010s | Mobile, IoT, AI/ML | Cloud SQL databases, distributed relational |
| 2020s | AI-native, serverless | Vector databases + relational integration |
