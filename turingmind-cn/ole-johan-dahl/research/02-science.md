# Ole-Johan Dahl - Scientific Contributions

## Object-Oriented Programming Origins

### The Problem: Simulation Complexity
- Need to model complex real-world systems
- Existing languages inadequate for discrete event simulation
- ALGOL 60 good but missing key abstraction mechanisms

### The Solution: Objects and Classes
- **Class concept:** Template for creating objects
- **Object concept:** Instance with data and procedures
- **Inheritance:** Code reuse and specialization
- **Encapsulation:** Data hiding and abstraction

## Simula Language Design

### Simula I (1962-1964)
Purpose: Discrete event simulation

Key innovations:
- **Process concept:** Active objects with lifecycle
- **Scheduling:** Quasi-parallel execution
- **Simulation-specific:** Time-based event handling

### Simula 67 (1965-1967)
Evolution to general-purpose language

Key features:
- **Unified class concept:** Generalized process idea
- **Prefixing mechanism:** Single inheritance
- **Virtual procedures:** Dynamic binding
- **Reference semantics:** Object identity
