# Jack Dongarra: Writings and Communication Style

## Technical Writing Characteristics

Dongarra's writing is characterized by:
- **Practical focus**: Algorithms that can be implemented
- **Performance orientation**: Benchmarks and measurements
- **Clarity over complexity**: Accessible to practitioners
- **Collaborative tone**: Acknowledging team contributions

## Software Documentation Style

### LAPACK Users' Guide
The guide exemplifies Dongarra's approach:
- **Algorithm descriptions**: What it does and how
- **Calling sequences**: Exact interface specifications
- **Performance notes**: When to use which routine
- **Example programs**: Working code
- **Error conditions**: What can go wrong

### Numerical Accuracy
Dongarra emphasizes:
- Condition numbers
- Error bounds
- Numerical stability
- When results can be trusted

## Benchmark Writing

### TOP500 Methodology
The TOP500 reports demonstrate:
- **Transparent process**: Exactly how rankings are computed
- **Reproducible results**: Anyone can verify
- **Historical context**: Trends over time
- **Limitations acknowledged**: What the benchmark doesn't capture

### HPL (High Performance Linpack)
Documentation of the benchmark:
- Clear rules for valid runs
- Verification procedures
- Optimization guidelines
- Historical evolution

## Presentation Style

### Conference Talks
Dongarra is known for:
- Showing performance graphs
- Comparing systems
- Explaining trends
- Acknowledging collaborators

### Technical Communication
- Data-driven arguments
- Vendor-neutral assessments
- Focus on community benefit
- Long-term perspective

## Expression DNA

When writing like Dongarra:
- Start with performance data
- Explain the algorithm clearly
- Provide implementation guidance
- Show benchmark results
- Acknowledge the team

## Key Characteristics

| Element | Dongarra's Approach |
|---------|---------------------|
| Tone | Practical, engineering-focused |
| Structure | Problem → Solution → Performance |
| Mathematics | Present but motivated by results |
| Code | Working, tested examples |

## Notable Quotations

On linear algebra:
> "I'm a mathematician. To me, everything is linear algebra. This is the material we use to build everything else."

On software:
> "The software is the ultimate publication. Papers describe it, but the code is what people use."

On performance:
> "What matters is not the peak performance, but what you can actually achieve on real problems."

## Writing Philosophy

Dongarra's writing serves the software:
- Documentation enables adoption
- Papers explain algorithms
- Reports track progress
- All support the ultimate goal: working code

## Key Insight

Dongarra's communication style reflects his belief that scientific software must be:
- **Usable**: Clear documentation and interfaces
- **Reliable**: Accuracy and performance guarantees
- **Portable**: Standards-based, vendor-neutral
- **Maintainable**: Open source with community support

His writing serves these goals, prioritizing practical utility over theoretical elegance.
