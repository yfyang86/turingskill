# Alan J. Perlis - Ethics and Philosophy

## Views on Programming as Discipline

### Programming as Intellectual Activity
- Believed programming requires mathematical thinking
- Advocated for programming literacy as fundamental education
- Argued against treating programming as mere clerical work
- Emphasized creativity in algorithm design

### Professional Responsibility
- Promoted high standards in software development
- Advocated for rigorous testing and verification
- Believed programmers should understand complete systems
- Supported formal methods when practical

## Educational Ethics

### Democratization of Computing
- Worked to make computing education accessible
- Founded departments to train diverse students
- Believed computer science should serve society
- Advocated for interdisciplinary approaches

### Teaching Philosophy
- Knowledge should be freely shared
- Programming skills empower individuals
- Computer science education shapes society
- Mentorship responsibility of senior researchers

## Technology and Society

### Human-Centered Computing
- Warned against technology for technology's sake
- Emphasized human factors in system design
- Advocated for usable programming languages
- Supported tools that enhance human capability

### Professional Standards
- Promoted ACM Code of Ethics
- Supported peer review and quality standards
- Advocated for honest reporting of results
- Believed in scientific integrity

## Controversial Positions

### GOTO Statement Debate
- Participated in structured programming discussions
- Acknowledged Dijkstra's concerns but pragmatic
- Believed good programmers use tools appropriately
- Opposed dogmatic approaches to programming

### Language Wars
- Advocated for ALGOL over commercial languages
- Criticized COBOL's verbosity
- Supported elegant, mathematical notation
- Valued language simplicity and power

## Legacy Values

- Intellectual honesty in research
- Excellence in education
- Service to computing community
- Balance of theory and practice
