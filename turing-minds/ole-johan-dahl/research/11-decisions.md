# Ole-Johan Dahl - Key Decisions and Their Impact

## Career-Defining Decisions

### 1. Joining NDRE (1952)
**Context:** Post-military service, early computing era
**Decision:** Norwegian Defense Research Establishment
**Impact:** Access to computers; met Nygaard; began computing career
**Alternative:** University path would have been different

### 2. Collaboration with Nygaard (1960)
**Context:** Both at NDRE, shared interests
**Decision:** Partner on simulation language
**Impact:** 40+ year collaboration; Simula; OOP creation
**Risk:** Partnership could have failed; instead transformative

### 3. Simula I Development (1962-1964)
**Context:** Need for simulation language
**Decision:** Extend ALGOL 60 with process concepts
**Impact:** First object-oriented concepts; simulation applications
**Challenge:** Implementation on limited hardware

### 4. Simula 67 Generalization (1965-1967)
**Context:** Simula I success; broader potential seen
**Decision:** Generalize to full programming language
**Impact:** Universal OOP concepts; influenced all future languages
**Insight:** Recognition that simulation ideas had general applicability

### 5. Move to University of Oslo (1968)
**Context:** NCC success; academic opportunity
**Decision:** Join university as professor
**Impact:** Built CS department; trained generations; research freedom
**Trade-off:** Less direct language development

### 6. Focus on Verification (1970s-1990s)
**Context:** Simula established; new interests
**Decision:** Research program verification
**Impact:** Textbook; formal methods contribution; education
**Evolution:** From language design to methodology

## Technical Decisions

### ALGOL 60 Base
**Decision:** Extend rather than create from scratch
**Impact:** Familiar syntax; easier adoption
**Advantage:** Built on solid foundation
**Risk:** Some constraints inherited

### Process/Class Concept
**Decision:** Unified concept for simulation objects
**Impact:** Core OOP innovation
**Evolution:** From simulation-specific to general
**Legacy:** Fundamental to modern programming

### Single Inheritance
**Decision:** Prefix classes, single inheritance
**Impact:** Simpler semantics
**Trade-off:** Less flexible than multiple inheritance
**Later:** Multiple inheritance added by others
