# Michael Stonebraker - Key Decisions

## Early Career Decisions

### 1971: Joining Berkeley
- **Context:** New Ph.D. from Michigan
- **Alternative:** Industry research labs
- **Decision:** Academic career at Berkeley
- **Opportunity:** Build research program
- **Outcome:** Founded database research group

### 1973: Starting INGRES Project
- **Context:** Relational theory emerging
- **Risk:** Relational databases unproven
- **Decision:** Build complete relational system
- **Team:** Graduate students
- **Result:** First production relational database

### 1975: INGRES Open Source Release
- **Decision:** Release source code publicly
- **Alternative:** Commercialize immediately
- **Reasoning:** Spread relational technology
- **Impact:** Educational use worldwide
- **Legacy:** Influenced database education

## Relational Database Era

### 1982: Relational Technology Inc.
- **Context:** INGRES commercial potential
- **Decision:** Co-found company
- **Partners:** Former students
- **Outcome:** Successful database company
- **Exit:** Acquired by Computer Associates

### 1984: Leaving INGRES
- **Context:** Commercial success achieved
- **Decision:** Return to research focus
- **Next:** Postgres project
- **Reasoning:** New research problems

## Postgres Era

### 1986: Starting Postgres
- **Motivation:** Extend relational model
- **Innovation:** Object-relational features
- **Risk:** Complex system challenge
- **Team:** New generation of students
- **Timeline:** Multi-year project

### 1995: Illustra Company
- **Context:** Postgres technology mature
- **Decision:** Commercialize Postgres
- **Company:** Illustra Information Technologies
- **Result:** Successful startup
- **Acquisition:** Bought by Informix

### 1996: Joining MIT
- **Context:** Berkeley to MIT move
- **Decision:** Accept MIT faculty position
- **Opportunity:** Fresh start, new resources
- **Challenge:** Building new research group
- **Outcome:** Continued productive research
